CONTEXTO DO PROJETO
───────────────────
Quero construir um CRM white-label para agências de viagens, para vender como
produto SaaS. Já tenho um CRM de assessoria de marketing funcionando (JP CRM)
e quero replicar a mesma arquitetura técnica e qualidade de código para o
segmento de turismo.

O produto final deve ser genérico o suficiente para qualquer agência de viagens,
com o nome da agência configurável (white-label).

──────────────────────────────────────────────────────────────

NÃO copiar as cores, fontes nem visual do JP CRM (aquele é terracota/dark).

ESTRUTURA DE MÓDULOS — CRM AGÊNCIA DE VIAGENS
──────────────────────────────────────────────

ABA 0 — DASHBOARD (painel inicial)
  KPIs do mês: leads ativos, em cotação, clientes/reservas ativos, receita de comissões
  Funil de conversão de leads (por etapa com % e barra)
  Gráfico de leads por dia da semana (HTML/CSS puro, sem bibliotecas)
  SLA de Follow-up: em dia / vencido / sem data + barra de % saúde
  Fila de Risco: leads sem retorno com follow-up vencido (dias de atraso)

ABA 1 — PIPELINE
  Kanban com colunas: Novo Lead → Cotação → Proposta Enviada → Reservado → Concluído | Cancelado
  Cards com: destino de interesse, período, nº de passageiros, orçamento estimado
  Botão "Gerar Cotação →" nos leads em Proposta para criar uma Reserva
  Badge de tipo: Lua de Mel | Família | Aventura | Negócios | Cruzeiro | Grupos

ABA 2 — CLIENTES (Viajantes)
  Dados pessoais: nome, empresa, email, WhatsApp, Instagram
  Documentos: CPF, passaporte (número + validade), data de nascimento
  Localização: cidade / estado / país
  Preferências: classe aérea, tipo de hotel, regime alimentar, acomodação
  Histórico de viagens (reservas vinculadas)
  Alerta automático: passaporte vencendo em menos de 6 meses
  Alerta: aniversário do cliente (oportunidade de venda)

ABA 3 — RESERVAS (módulo central)
  Dados da viagem:
    - Destino(s), datas de ida e volta
    - Tipo: Pacote | Só Aéreo | Só Hotel | Cruzeiro | Rodoviário | Transfer
    - Nº de passageiros adultos / crianças
    - Status: Cotação → Confirmado → Pago → Em Viagem → Concluído | Cancelado
  Financeiro da reserva:
    - Valor total cobrado ao cliente
    - Custo (repasse aos fornecedores)
    - Comissão da agência (valor total - custo)
    - Margem %
    - Parcelas do cliente (quantidade, valor, datas, status pago/pendente/vencido)
    - Recorrência de pagamento: Mensal | Parcelado | Pagamento Único (ver padrão 13)
    - Dia fixo de vencimento das parcelas (ver padrão 14)
    - Status efetivo calculado em runtime, sem gravar "vencido" no banco (ver padrão 15)
  Fornecedores utilizados (multi: hotel + aéreo + transfer + seguro)
  Prazo de cancelamento sem multa (alerta automático)
  Passageiros vinculados (lista dos clientes na reserva)
  Documentos: número de reserva, localizador, voucher
  Botão Editar disponível diretamente na lista de reservas e na lista de parcelas

ABA 4 — FORNECEDORES
  Tipo: Hotel | Operadora | Companhia Aérea | Locadora | Seguro | Transfer | Outro
  Contato comercial
  % de comissão padrão
  Prazo de cancelamento sem multa (dias antes)
  Destinos/rotas que operam
  Histórico de reservas utilizando este fornecedor

ABA 5 — FINANCEIRO
  Período selecionável (mês/ano)
  Balanço anual acumulado
  Cards do mês:
    - Receita (comissões de reservas concluídas)
    - A Receber (comissões confirmadas ainda não recebidas)
    - Despesas operacionais
    - Saldo do mês
  Meta mensal com barra de progresso
  Tabela de comissões por reserva no período
  Tabela de parcelas de clientes (vencendo / vencidas / pagas)
    - Status calculado em runtime via statusEfetivo() (padrão 15)
    - Data de vencimento calculada via datVencPagamento() (padrão 14)
    - Botão Editar disponível em cada linha da tabela de cobranças
  Alertas de parcelas vencidas automaticamente por dia de vencimento
  Despesas da agência (ferramentas, aluguel, etc.)
    - Despesas mensais com data_fim para encerramento sem perda de histórico (padrão 17)
    - Botão Editar disponível em cada despesa
    - Despesas mensais só aparecem a partir do mês de inclusão

ABA 6 — ALERTAS (painel de avisos automáticos)
  🛂 Passaportes vencendo em menos de 6 meses
  ✈️ Viagens saindo nos próximos 7 dias
  ⏰ Prazos de cancelamento se aproximando (< 72h)
  💳 Parcelas de clientes vencendo hoje/amanhã
  🎂 Aniversários desta semana (oportunidade de venda)
  📋 Reservas sem pagamento confirmado


SCHEMA SUPABASE — SQL COMPLETO
────────────────────────────────
Rodar no SQL Editor do Supabase na ordem abaixo:

-- 1. CLIENTES (viajantes)
create table clientes (
  id uuid primary key default gen_random_uuid(),
  nome text not null,
  email text,
  whatsapp text,
  instagram text,
  empresa text,
  cpf text,
  passaporte text,
  passaporte_validade date,
  data_nascimento date,
  cidade text,
  estado text,
  pais text default 'Brasil',
  classe_aerea text default 'Econômica',
  tipo_hotel text,
  regime_alimentar text,
  preferencias text,
  como_conheceu text,
  observacao text,
  status text default 'Ativo',
  av text,
  created_at timestamptz default now()
);

-- 2. FORNECEDORES
create table fornecedores (
  id uuid primary key default gen_random_uuid(),
  nome text not null,
  tipo text, -- Hotel | Operadora | Aérea | Locadora | Seguro | Transfer | Outro
  contato_nome text,
  contato_email text,
  contato_whatsapp text,
  comissao_pct numeric default 0,
  prazo_cancelamento_dias integer,
  destinos text,
  observacao text,
  status text default 'Ativo',
  created_at timestamptz default now()
);

-- 3. LEADS
create table leads (
  id uuid primary key default gen_random_uuid(),
  nome text not null,
  whatsapp text,
  email text,
  destino_interesse text,
  periodo_pretendido text,
  tipo_viagem text, -- Lua de Mel | Família | Aventura | Negócios | Cruzeiro | Grupos
  num_passageiros integer default 1,
  orcamento_estimado numeric,
  status text default 'novo', -- novo | cotacao | proposta | reservado | concluido | cancelado
  origem text, -- indicação | instagram | google | site | feiras | outro
  followup date,  -- data do próximo follow-up (usado no Dashboard SLA)
  obs text,
  contato_data text,
  cliente_id uuid references clientes(id) on delete set null,
  reserva_id uuid, -- será preenchido ao converter
  created_at timestamptz default now()
);

-- 4. RESERVAS (central)
create table reservas (
  id uuid primary key default gen_random_uuid(),
  codigo text, -- número da reserva (ex: RES-001)
  cliente_id uuid references clientes(id) on delete set null,
  lead_id uuid references leads(id) on delete set null,
  destino text not null,
  destinos_multiplos text, -- para roteiros com vários destinos
  tipo text, -- Pacote | Aéreo | Hotel | Cruzeiro | Rodoviário | Transfer
  data_ida date,
  data_volta date,
  num_adultos integer default 1,
  num_criancas integer default 0,
  status text default 'cotacao', -- cotacao | confirmado | pago | em_viagem | concluido | cancelado
  valor_total numeric default 0,
  custo_total numeric default 0,
  comissao numeric generated always as (valor_total - custo_total) stored,
  margem_pct numeric generated always as (
    case when valor_total > 0 then round(((valor_total - custo_total)/valor_total)*100, 2) else 0 end
  ) stored,
  localizador text,
  prazo_cancelamento date,
  observacao text,
  created_at timestamptz default now()
);

-- 5. PASSAGEIROS DA RESERVA (N clientes por reserva)
create table reserva_passageiros (
  id uuid primary key default gen_random_uuid(),
  reserva_id uuid references reservas(id) on delete cascade,
  cliente_id uuid references clientes(id) on delete cascade,
  tipo text default 'Adulto' -- Adulto | Criança | Bebê
);

-- 6. ITENS DA RESERVA (serviços incluídos)
create table reserva_itens (
  id uuid primary key default gen_random_uuid(),
  reserva_id uuid references reservas(id) on delete cascade,
  fornecedor_id uuid references fornecedores(id) on delete set null,
  tipo text, -- Aéreo | Hotel | Transfer | Seguro | Passeio | Outro
  descricao text,
  valor_custo numeric default 0,
  valor_venda numeric default 0,
  status text default 'pendente', -- pendente | confirmado | cancelado
  localizador text,
  created_at timestamptz default now()
);

-- 7. PARCELAS (pagamentos do cliente)
create table parcelas (
  id uuid primary key default gen_random_uuid(),
  reserva_id uuid references reservas(id) on delete cascade,
  cliente_id uuid references clientes(id) on delete cascade,
  numero integer,
  valor numeric not null,
  data_vencimento date,
  data_pagamento date,
  status text default 'pendente', -- pendente | pago | vencido
  forma_pagamento text, -- PIX | Cartão | Transferência | Dinheiro
  observacao text,
  -- campos para recorrência avançada:
  recorrencia text default 'Parcelado', -- Mensal | Parcelado | Pagamento Único
  dia_vencimento integer,               -- dia fixo (1-31) para calcular vencimento por mês
  num_parcelas integer                  -- total de parcelas (quando recorrência = Parcelado)
);

-- 8. COMISSÕES (o que a agência recebe dos fornecedores)
create table comissoes (
  id uuid primary key default gen_random_uuid(),
  reserva_id uuid references reservas(id) on delete cascade,
  fornecedor_id uuid references fornecedores(id) on delete set null,
  valor numeric not null,
  data_prevista date,
  data_recebimento date,
  status text default 'pendente', -- pendente | recebido | cancelado
  observacao text
);

-- 9. DESPESAS OPERACIONAIS
create table despesas (
  id uuid primary key default gen_random_uuid(),
  item text not null,
  valor numeric,
  periodo text, -- Mensal | Único
  categoria text,
  data date,      -- data de início (ou data única)
  data_fim date,  -- null = ativa; preenchida = encerrada (soft-delete para mensais)
  created_at timestamptz default now()
);

-- 10. ALERTAS (gerados automaticamente ou manuais)
create table alertas (
  id uuid primary key default gen_random_uuid(),
  tipo text, -- passaporte | aniversario | cancelamento | parcela | viagem
  titulo text,
  descricao text,
  cliente_id uuid references clientes(id) on delete cascade,
  reserva_id uuid references reservas(id) on delete cascade,
  data_alerta date,
  lido boolean default false,
  created_at timestamptz default now()
);

  ↓

  ↓

  ↓

  ↓
